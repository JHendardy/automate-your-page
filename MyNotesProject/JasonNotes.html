Unit 0

The Web

- made up of HTML pages
- invented in early 1990s

Some Key Terms

HTML - hypertext markup language
URLs - uniform resource locator
HTTP - hypertext transfer protocol
Web Application - any software that runs in the web browser

HTML Tags & Elements

- The structure of an element looks like this:

<tag attr="value"> contents </tag>

Anchor tags are used to include links. They look and work like this:

<a>
<a href="www.example.com">example</a>

Images can be displayed with the image tag. They are a void tag because they do not have closing tags. It's good to add an alt attribute to help if the image does not load and if people with impaired vision are visiting the site with a site reader. They look and work like this:

<img>
<img src="url" alt="text">

Inline vs Block Elements

- HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

- Some examples of inline tags are: a, br, em, img, span

- Some examples of block tags are: div, h1, p

Structure of HTML Page

The structure of an HTML page should look like:

<!DOCTYPE HTML>
<html>
<head>
</head>
<body>
</body>
</html>

- In addition to this structure, structured code refers to the layout of code. It refers to the DOM, or the branches of the HTML.

- Semantics within HTML is the practice of giving content on the page meaning and structure by using the proper element. Semantic code describes the value of content on a page, regardless of the style or appearance of that content.

Stages of Learning

- There are four stages of learning and it takes years to get to fluency. The stages are:

0 - Ignorance
1 - Awareness
2 - Ability
3 - Fluency

Some Key Terms and Ideas

HTML - is used for structure
CSS - or cascading style sheets is used for style
DOM - or document object model refers to the HTML tree, which one can see in dev tools (such as Google's Developer Tools in Chrome)

- Everything is a box. It's helpful to break down webpages into seperate boxes or divs to help visualize the code.

- DIVs need classes to help label them. This will help when adding CSS to certain divs or elements.

- Avoid repetition to make your life easier.

- DRY = Don't Repeat Yourself. This is important and helpful for organization and readable code. By being organized and not repeating code it's easier to edit and figure out an issue with a project.

- Group reusable css properties together
- Name these groups logically
- Add your selectors to the various css groups

Some CSS Things to Know

- If you want to select a class you can use ."classname" in the CSS sheet. For IDs use #.

Margin - outer layer
Border - border inside margin but around padding
Padding - gutter around content
Content - very center

- Since DIVs are block elements (as opposed to inline), by default they take up the entire width of a page.

- Adding the rule display: flex to the appropriate CSS wil override this behavior and let DIVs apppear next to each other

box-sizing: border-box

- Makes it so the element size include border and padding, size of the box will always stay the same when border and padding is tweaked. Kind of like holding SHIFT in PSD.

- Because it's new you need to make sure browsers are compatible.

- For graceful degradation one must account for all browsers:

-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;

Computer Science

- Computer Science is about how to solve problems by breaking them down into smaller pieces and precisely and mechanincally describing a sequence of steps to solve each piece and can be executed by a computer.

- Programming is the core of computer science

Some Key Terms

- computer - universal machine that can be programmed to do any computation

- program - precise sequence of steps that a computer can follow to do something useful. Web browsers, games, mobile apps, and simple print statements are all examples of computer programs.

- precise sequence of steps - computer itself doesn't know how to do anything. This is important to make a program useful.

- computation - technically it is the action of mathematical calculation, or in the sense of programming it's the act of computing

- high-level language - like Python

- input - as in information fed into a program, which then is run by the computer

- interpreter - (like Python) runs the program, interprets it, and executes the program in a language the computer can understand

- procedural thinking - for example "...follows the instructions in your code" is a way to think like a programmer

- abstract thinking - an example would be thinking of a Python program, a Python interpreter, and a web browser as different versions of the same thing (a computer program)

- Compiler - a program that produces other programs. It does all the work at once and then runs the program. An interpreter does this all tat the same time.

Natural Languages Vs Computer Programming Language

- Natural languages have ambiguity. People understand things in different ways.
- Natural languages are also very verbose
- Computer programming languages allow for shorter commands / lines of codes to make the computer do a precise thing.

Variables

Strings

- A sequence of characters surronded by quotes.

- Without quotes the string looks and will be interpreted as a variable.

- <string> + <string> is the concatenatin of the two strings.

- We can not add number and strings together, but we can multiply them.

- One thing that we can do with strings that we can't do with numbers is extract subsequences.

- In addition to single quotes or double quotes to create a string, you can also you TRIPLE quotes to create a multi-line string.

- Indexing Strings:
<string>[<expression>]

- Selecting Sub-sequences:
<string>[<expression>:<expression>]

What is the difference between 2 + 2 and "2" + "2"?

In Python, 2 is a number while "2" is a string.

The code 2 + 2 would give 4.

The code "2" + "2" would give "22".

Find

- A method. A built in procedure in Python.

- Will output the number that gives first position in search string where the target string appears.

- If target string is not found, the output will be -1

- Parameters are what goes inside the parentheses.

- example: "text".find("t", 1)
- "t" and 1 are parameters

Functions

- Or sometimes referred to as Procedures.

- Functions are made by starting a line of code with the keyword def and then giving a function name followed by the function parameters in parentheses. These parameters will eventually be replaced by actual values when the function is used (called).

- Functions are tools that programmers can create and reuse forever! Once you've defined a function once, you never have to define it again.

- Structure wise they look like:
def parameter(function)

- Example:
def say_hello(name)
greeting = "Hello" + name + "!"
return greeting

print say_hello("Miriam")
print say_hello("Andy")

- The return keyword tells Python exactly what the function should produce as output. If a function doesn't have a return statement, then the interpreter will print None.
